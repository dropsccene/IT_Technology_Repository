## 三、关键问题：怎么把两个表联系起来？🔗

### 思考：
```
┌─────────────────────────────────────────────────────────────┐
│                      连接的关键                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
```

## 四、逐步构建 SQL 语句 🏗️

### 第1步：先找出浙江大学的用户
```sql
SELECT device_id
FROM user_profile
WHERE university = '浙江大学';
```
**结果：** device_id = 2315

---

### 第2步：再找这个用户的答题记录
```sql
SELECT *
FROM question_practice_detail
WHERE device_id = 2315;
```
**结果：** 3 条答题记录（题号 115、116、117）

---

### 第3步：把两步合并成一条 SQL（用 JOIN 连接）
```sql
SELECT q.*
FROM question_practice_detail AS q
JOIN user_profile AS u
ON q.device_id = u.device_id
WHERE u.university = '浙江大学';
```

## 五、完整代码 ✅

```sql
SELECT q.device_id, 
       q.question_id, 
       q.result
FROM question_practice_detail AS q
JOIN user_profile AS u
ON q.device_id = u.device_id
WHERE u.university = '浙江大学';
```

### 或者更简洁的写法：
```sql
SELECT device_id, 
       question_id, 
       result
FROM question_practice_detail
WHERE device_id IN (
    SELECT device_id 
    FROM user_profile 
    WHERE university = '浙江大学'
);
```

## 六、代码详解 🔍

### 方法1：JOIN 连接（推荐）

```sql
-- 第1部分：从答题记录表开始
FROM question_practice_detail AS q

-- 第2部分：连接用户信息表
JOIN user_profile AS u

-- 第3部分：连接条件（通过 device_id 连接）
ON q.device_id = u.device_id

-- 第4部分：筛选浙江大学的用户
WHERE u.university = '浙江大学'

-- 第5部分：选择要显示的字段
SELECT q.device_id, q.question_id, q.result
```

**执行流程：**
```
┌─────────────────────────────────────────────────────────────┐
│                    JOIN 执行流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1️⃣ 从 question_practice_detail 表取所有答题记录              │
│     （7 条记录）                                             │
│                                                             │
│  2️⃣ 和 user_profile 表通过 device_id 连接                     │
│     （每条答题记录都找到对应的用户信息）                       │
│                                                             │
│  3️⃣ 用 WHERE 筛选 university = '浙江大学' 的记录              │
│     （只保留浙江大学的用户）                                 │
│                                                             │
│  4️⃣ 返回 device_id、question_id、result 三个字段              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 九、新手常见疑问 ❓

### Q1：为什么要用 JOIN？不能直接查吗？
**A：** 因为信息在两个表里！
- 大学信息在 user_profile 表
- 答题信息在 question_practice_detail 表
- 必须把两个表"连"起来才能同时获取

### Q2：device_id 是什么？
**A：** 设备的唯一标识，就像身份证号，用来识别是哪个用户

### Q3：为什么不用 id 连接？
**A：** 因为两个表的 id 含义不同！
- user_profile 的 id 是用户记录编号
- question_practice_detail 的 id 是答题记录编号
- 只有 device_id 在两个表里代表同一个东西

### Q4：AS q 和 AS u 是什么意思？
**A：** 给表起别名，让代码更简洁
- `question_practice_detail AS q` → 后面可以用 q 代替长表名
- `user_profile AS u` → 后面可以用 u 代替长表名
