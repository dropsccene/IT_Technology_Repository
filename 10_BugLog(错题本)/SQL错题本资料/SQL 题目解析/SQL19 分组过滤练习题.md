# ❌ 代码有逻辑错误！WHERE 不能过滤聚合结果

---

## 一、核心错误分析 🔍

| 错误点 | 你的代码 | 正确写法 | 原因 |
|-------|---------|---------|------|
| **过滤时机错误** | `WHERE (question_cnt) < 5` | `HAVING AVG(question_cnt) < 5` | **WHERE 过滤行，HAVING 过滤组** |
| **聚合函数位置** | 在 WHERE 中不能用 AVG | 在 HAVING 中用 AVG | WHERE 执行时还没分组 |
| **过滤对象** | 过滤单个用户 | 过滤学校（组） | 题目要求按学校平均値筛选 |

### ❌ 你的代码逻辑：
```sql
WHERE (question_cnt) < 5  -- 先找出发帖数<5 的【用户】
GROUP BY university       -- 再按学校分组
```
**结果：** 统计的是"发帖少的那些用户"所在的学校平均值，而不是"所有用户"的学校平均值！

### ✅ 正确逻辑：
```sql
GROUP BY university       -- 先按学校分组
HAVING AVG(...) < 5       -- 再筛选平均发帖数<5 的【学校】
```

## 二、正确代码 ✅

```sql
SELECT university,
       ROUND(AVG(question_cnt), 3) AS avg_question_cnt,
       ROUND(AVG(answer_cnt), 3) AS avg_answer_cnt
FROM user_profile
GROUP BY university
HAVING AVG(question_cnt) < 5 OR AVG(answer_cnt) < 20;
```

## 四、WHERE vs HAVING 核心区别 ⚠️

```
┌─────────────────────────────────────────────────────────────┐
│                    WHERE vs HAVING 对比                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌───────────────┬─────────────────┬─────────────────┐     │
│  │     特性      │      WHERE      │      HAVING     │     │
│  ├───────────────┼─────────────────┼─────────────────┤     │
│  │  执行时机     │  GROUP BY 之前  │  GROUP BY 之后  │     │
│  ├───────────────┼─────────────────┼─────────────────┤     │
│  │  过滤对象     │    原始数据行    │    分组后的组    │     │
│  ├───────────────┼─────────────────┼─────────────────┤     │
│  │  聚合函数     │   ❌ 不能使用    │   ✅ 可以使用    │     │
│  ├───────────────┼─────────────────┼─────────────────┤     │
│  │  使用场景     │  筛选用户/记录   │  筛选学校/分组   │     │
│  └───────────────┴─────────────────┴─────────────────┘     │
│                                                             │
│  本题场景：                                                  │
│  要筛选"平均发帖数<5 的学校" → 这是分组后的结果 → 用 HAVING  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 五、数据验证（为什么你的代码错了）📊

### 原始数据：
| id | university | question_cnt | answer_cnt |
|----|------------|--------------|------------|
| 1 | 北京大学 | 2 | 12 |
| 3 | 北京大学 | 3 | 30 |
| 2 | 复旦大学 | 5 | 25 |
| 7 | 复旦大学 | 6 | 52 |
| 4 | 浙江大学 | 1 | 2 |
| 5 | 山东大学 | 15 | 70 |
| 6 | 山东大学 | 7 | 13 |

### ❌ 你的代码执行过程（先 WHERE 后 GROUP）：
```sql
WHERE question_cnt < 5 OR answer_cnt < 20
```
**筛选后的行：**
| id | university | question_cnt | answer_cnt | 命中条件 |
|----|------------|--------------|------------|---------|
| 1 | 北京大学 | 2 | 12 | 都命中 |
| 3 | 北京大学 | 3 | 30 | question<5 |
| 4 | 浙江大学 | 1 | 2 | 都命中 |
| 6 | 山东大学 | 7 | 13 | answer<20 |

**然后 GROUP BY university 计算平均值：**
- 北京大学：(2+3)/2 = 2.5, (12+30)/2 = 21 → **保留**
- 浙江大学：1, 2 → **保留**
- 山东大学：7, 13 → **保留**
- 复旦大学：被 WHERE 过滤掉了，**消失！**

### ✅ 正确代码执行过程（先 GROUP 后 HAVING）：
```sql
GROUP BY university → 计算所有学校的平均值
```
**分组后的结果：**
| university | avg_question | avg_answer |
|------------|--------------|------------|
| 北京大学 | (2+3)/2 = 2.5 | (12+30)/2 = 21 |
| 复旦大学 | (5+6)/2 = 5.5 | (25+52)/2 = 38.5 |
| 浙江大学 | 1 | 2 |
| 山东大学 | (15+7)/2 = 11 | (70+13)/2 = 41.5 |

**然后 HAVING 筛选：**
- 北京大学：2.5 < 5 ✅ **保留**
- 复旦大学：5.5≮5 且 38.5≮20 ❌ 排除
- 浙江大学：1 < 5 ✅ **保留**
- 山东大学：11≮5 且 41.5≮20 ❌ 排除

**最终结果：** 北京大学、浙江大学

## 七、易错点总汇 ⚠️

| 易错点 | 错误写法 | 正确写法 |
|-------|---------|---------|
| 聚合函数在 WHERE | `WHERE AVG(q) < 5` | `HAVING AVG(q) < 5` |
| 缺少 HAVING | 直接用 WHERE 筛选平均值 | 必须用 HAVING |
| 逻辑运算符 | `AND` | 题目说"或" → `OR` |
| 小数位数 | `ROUND(..., 1)` | 题目要求 3 位 → `ROUND(..., 3)` |
| 列别名 | 无别名 | `AS avg_question_cnt` |

## 八、记忆口诀 📖

```
┌────────────────────────────────────────────────┐
│  WHERE 和 HAVING 的区别记忆法：                  │
│                                                │
│  WHERE 是"事前筛选"                            │
│  → 分组前，挑 rows，不能用聚合                  │
│  → 比如：只要男性用户 WHERE gender='male'       │
│                                                │
│  HAVING 是"事后筛选"                            │
│  → 分组后，挑 groups，可以用聚合                │
│  → 比如：只要平均发帖<5 的学校                  │
│     HAVING AVG(question) < 5                   │
│                                                │
│  口诀：先 WHERE 后 GROUP，HAVING 跟在 GROUP 后！  │
└────────────────────────────────────────────────┘
```

## 九、完整模板 📝

```sql
SELECT 分组字段，
       聚合函数 1 AS 别名 1,
       聚合函数 2 AS 别名 2
FROM 表名
WHERE 行筛选条件          -- ⚠️ 可选，不能用聚合函数
GROUP BY 分组字段
HAVING 组筛选条件          -- ⚠️ 可选，可以用聚合函数
ORDER BY 排序字段;         -- ⚠️ 可选
```

**本题套用：**
```sql
SELECT university,
       ROUND(AVG(question_cnt), 3) AS avg_question_cnt,
       ROUND(AVG(answer_cnt), 3) AS avg_answer_cnt
FROM user_profile
-- 没有 WHERE 条件
GROUP BY university
HAVING AVG(question_cnt) < 5 OR AVG(answer_cnt) < 20;
```

**总结：你的代码主要问题是混淆了 WHERE 和 HAVING 的使用场景！** 记住：**过滤聚合结果（平均值、总和等）必须用 HAVING！** 👍
