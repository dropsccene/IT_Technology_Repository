## 二、数据查询关键字（DQL）🔍

### 1️⃣ 基础查询（🔴 核心必学）

| 关键字        | 功能   | 示例                           | 易错点 ⚠️      |
| ---------- | ---- | ---------------------------- | ----------- |
| `SELECT`   | 选择字段 | `SELECT * FROM table`        | 不能省略        |
| `FROM`     | 指定表  | `FROM user_profile`          | 表名要正确       |
| `WHERE`    | 条件筛选 | `WHERE age > 18`             | **不能用聚合函数** |
| `AS`       | 列别名  | `AS 最高GPA`                   | 可省略但建议写     |
| `DISTINCT` | 去重   | `SELECT DISTINCT university` | 放在SELECT后   |
| `LIMIT`    | 限制条数 | `LIMIT 10`                   | MySQL特有     |
| `ORDER BY` | 排序   | `ORDER BY gpa DESC`          | **默认ASC升序** |
| `ASC`      | 升序   | `ORDER BY gpa ASC`           | 可省略         |
| `DESC`     | 降序   | `ORDER BY gpa DESC`          | 必须显式写       |

### 2️⃣ 分组聚合（🔴 核心必学）

| 关键字        | 功能    | 示例                     | 易错点 ⚠️                 |
| ---------- | ----- | ---------------------- | ---------------------- |
| `GROUP BY` | 分组    | `GROUP BY university`  | **SELECT中非聚合字段必须在此出现** |
| `HAVING`   | 分组后筛选 | `HAVING COUNT(*) > 10` | **不能用WHERE替代**         |
| `MAX()`    | 最大值   | `MAX(gpa)`             | 聚合函数                   |
| `MIN()`    | 最小值   | `MIN(age)`             | 聚合函数                   |
| `SUM()`    | 求和    | `SUM(score)`           | 聚合函数                   |
| `AVG()`    | 平均值   | `AVG(gpa)`             | 聚合函数                   |
| `COUNT()`  | 计数    | `COUNT(*)`             | `COUNT(*)`包含NULL       |

### 3️⃣ 连接查询（🟡 进阶常用）

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`JOIN` / `INNER JOIN`|内连接|`JOIN table2 ON id = id`|**只返回匹配行**|
|`LEFT JOIN`|左连接|`LEFT JOIN table2`|**返回左表全部行**|
|`RIGHT JOIN`|右连接|`RIGHT JOIN table2`|返回右表全部行|
|`FULL JOIN`|全连接|`FULL JOIN table2`|MySQL不支持|
|`CROSS JOIN`|交叉连接|`CROSS JOIN table2`|笛卡尔积|
|`ON`|连接条件|`ON t1.id = t2.id`|**不能用WHERE替代**|
|`USING`|同名字段连接|`USING (id)`|字段名必须相同|

### 4️⃣ 集合运算（🟡 进阶常用）

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`UNION`|并集（去重）|`UNION`|**自动去重**|
|`UNION ALL`|并集（不去重）|`UNION ALL`|**性能更好**|
|`INTERSECT`|交集|`INTERSECT`|MySQL不支持|
|`EXCEPT`|差集|`EXCEPT`|MySQL不支持|

## 三、数据操作关键字（DML）✏️

### 1️⃣ 插入数据（🔴 核心必学）

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`INSERT INTO`|插入数据|`INSERT INTO table`|表名要正确|
|`VALUES`|指定值|`VALUES (1, 'name')`|**字段数要匹配**|
|`SET`|设置值|`SET col = value`|INSERT中可用|

```sql
-- 标准插入
INSERT INTO user_profile (id, device_id, gender, university, gpa)
VALUES (1, 2138, 'male', '北京大学', 3.4);

-- 多行插入
INSERT INTO user_profile VALUES (1, ...), (2, ...), (3, ...);

-- 插入并忽略冲突
INSERT IGNORE INTO table VALUES (...);

-- 插入冲突时更新
INSERT INTO table VALUES (...) ON DUPLICATE KEY UPDATE col = value;
```

### 2️⃣ 更新数据（🔴 核心必学）

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`UPDATE`|更新表|`UPDATE table`|**必须加WHERE**|
|`SET`|设置新值|`SET gpa = 3.8`|多个用逗号分隔|

```sql
-- 单字段更新
UPDATE user_profile SET gpa = 3.8 WHERE id = 1;

-- 多字段更新
UPDATE user_profile SET gpa = 3.8, age = 20 WHERE id = 1;

-- ⚠️ 危险：不加WHERE会更新全部行！
UPDATE user_profile SET gpa = 0;  -- 所有行gpa都变成0！
```

### 3️⃣ 删除数据（🔴 核心必学）

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`DELETE`|删除行|`DELETE FROM table`|**必须加WHERE**|
|`TRUNCATE`|清空表|`TRUNCATE TABLE table`|**不能回滚**|

```sql
-- 删除指定行
DELETE FROM user_profile WHERE id = 1;

-- ⚠️ 危险：不加WHERE会删除全部行！
DELETE FROM user_profile;  -- 所有行都被删除！

-- 清空表（更快，不能回滚）
TRUNCATE TABLE user_profile;
```

## 四、数据定义关键字（DDL）🏗️

### 1️⃣ 表操作（🟡 进阶常用）

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`CREATE TABLE`|创建表|`CREATE TABLE t (...)`|字段定义要完整|
|`ALTER TABLE`|修改表|`ALTER TABLE t ADD col`|语法因操作而异|
|`DROP TABLE`|删除表|`DROP TABLE t`|**数据永久删除**|
|`RENAME TABLE`|重命名表|`RENAME TABLE t1 TO t2`|两表名都要写|
|`IF EXISTS`|存在检查|`DROP TABLE IF EXISTS t`|避免报错|
|`IF NOT EXISTS`|不存在检查|`CREATE TABLE IF NOT EXISTS`|避免重复创建|

```sql
-- 创建表
CREATE TABLE user_profile (
    id INT PRIMARY KEY,
    device_id INT,
    gender VARCHAR(10),
    university VARCHAR(50),
    gpa DECIMAL(3,1)
);

-- 添加列
ALTER TABLE user_profile ADD COLUMN age INT;

-- 修改列
ALTER TABLE user_profile MODIFY COLUMN gpa DECIMAL(4,2);

-- 删除列
ALTER TABLE user_profile DROP COLUMN age;

-- 删除表
DROP TABLE IF EXISTS user_profile;
```

### 2️⃣ 索引操作（🟢 高级扩展）

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`CREATE INDEX`|创建索引|`CREATE INDEX idx ON t(col)`|索引名要唯一|
|`DROP INDEX`|删除索引|`DROP INDEX idx ON t`|要指定表名|
|`UNIQUE`|唯一索引|`UNIQUE KEY (col)`|值不能重复|
|`PRIMARY KEY`|主键|`PRIMARY KEY (id)`|每表只能一个|
|`FOREIGN KEY`|外键|`FOREIGN KEY (id) REFERENCES`|关联表必须存在|

### 3️⃣ 视图操作（🟢 高级扩展）

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`CREATE VIEW`|创建视图|`CREATE VIEW v AS SELECT...`|视图是虚拟表|
|`DROP VIEW`|删除视图|`DROP VIEW v`|不影响原表|
|`OR REPLACE`|替换视图|`CREATE OR REPLACE VIEW`|存在则替换|

## 五、数据控制关键字（DCL）🔐

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`GRANT`|授予权限|`GRANT SELECT ON t TO user`|权限要明确|
|`REVOKE`|撤销权限|`REVOKE SELECT ON t FROM user`|权限要匹配|
|`ALL PRIVILEGES`|所有权限|`GRANT ALL PRIVILEGES`|**谨慎使用**|

```sql
-- 授予查询权限
GRANT SELECT ON user_profile TO 'user1'@'localhost';

-- 授予所有权限
GRANT ALL PRIVILEGES ON database.* TO 'user1'@'localhost';

-- 撤销权限
REVOKE SELECT ON user_profile FROM 'user1'@'localhost';
```

## 六、事务控制关键字（TCL）💾

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`START TRANSACTION`|开始事务|`START TRANSACTION`|事务起点|
|`BEGIN`|开始事务|`BEGIN`|同上|
|`COMMIT`|提交事务|`COMMIT`|**永久保存**|
|`ROLLBACK`|回滚事务|`ROLLBACK`|**撤销所有更改**|
|`SAVEPOINT`|保存点|`SAVEPOINT sp1`|可回滚到此点|
|`AUTOCOMMIT`|自动提交|`SET AUTOCOMMIT = 0`|默认开启|

```sql
-- 完整事务示例
START TRANSACTION;
UPDATE account SET balance = balance - 100 WHERE id = 1;
UPDATE account SET balance = balance + 100 WHERE id = 2;
-- 如果出错
ROLLBACK;
-- 如果成功
COMMIT;
```

## 七、条件与逻辑运算符 🔣

### 1️⃣ 比较运算符（🔴 核心必学）

|运算符|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`=`|等于|`WHERE age = 18`|**不是==**|
|`!=` 或 `<>`|不等于|`WHERE age != 18`|两种写法都可|
|`>`|大于|`WHERE gpa > 3.5`|-|
|`<`|小于|`WHERE gpa < 3.5`|-|
|`>=`|大于等于|`WHERE gpa >= 3.5`|-|
|`<=`|小于等于|`WHERE gpa <= 3.5`|-|

### 2️⃣ 逻辑运算符（🔴 核心必学）

|运算符|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`AND`|与|`WHERE age > 18 AND gender = 'male'`|**优先级高于OR**|
|`OR`|或|`WHERE age > 18 OR age < 10`|优先级低于AND|
|`NOT`|非|`WHERE NOT age > 18`|可写为`age <= 18`|

### 3️⃣ 特殊条件（🟡 进阶常用）

|关键字|功能|示例|易错点 ⚠️|
|---|---|---|---|
|`IN`|在集合中|`WHERE id IN (1,2,3)`|**替代多个OR**|
|`NOT IN`|不在集合中|`WHERE id NOT IN (1,2,3)`|**NULL值要小心**|
|`BETWEEN`|范围内|`WHERE age BETWEEN 18 AND 25`|**包含边界值**|
|`LIKE`|模糊匹配|`WHERE name LIKE '张%'`|`%`是通配符|
|`IS NULL`|是空值|`WHERE age IS NULL`|**不能用= NULL**|
|`IS NOT NULL`|非空值|`WHERE age IS NOT NULL`|**不能用!= NULL**|
|`EXISTS`|存在检查|`WHERE EXISTS (SELECT...)`|子查询用|
|`REGEXP`|正则匹配|`WHERE name REGEXP '^张'`|MySQL特有|

```sql
-- IN 替代多个 OR
WHERE id IN (1, 2, 3)  -- 等价于 id = 1 OR id = 2 OR id = 3

-- LIKE 通配符
WHERE name LIKE '张%'      -- 张开头
WHERE name LIKE '%三'      -- 三结尾
WHERE name LIKE '%小%'     -- 包含小
WHERE name LIKE '张_'      -- 张+任意1字符

-- ⚠️ NULL 判断易错点
WHERE age = NULL    -- ❌ 错误！永远返回false
WHERE age IS NULL   -- ✅ 正确

-- BETWEEN 包含边界
WHERE age BETWEEN 18 AND 25  -- 等价于 age >= 18 AND age <= 25
```

## 十、易错点总汇 ⚠️

### 🔴 最高频错误TOP 10

|排名|错误类型|错误写法|正确写法|
|---|---|---|---|
|1|NULL判断|`WHERE age = NULL`|`WHERE age IS NULL`|
|2|WHERE用聚合|`WHERE MAX(gpa) > 3`|`HAVING MAX(gpa) > 3`|
|3|忘记GROUP BY|`SELECT uni, MAX(gpa)`|`SELECT uni, MAX(gpa) GROUP BY uni`|
|4|UPDATE无WHERE|`UPDATE t SET gpa = 0`|`UPDATE t SET gpa = 0 WHERE id = 1`|
|5|DELETE无WHERE|`DELETE FROM t`|`DELETE FROM t WHERE id = 1`|
|6|字符串无引号|`WHERE uni = 复旦大学`|`WHERE uni = '复旦大学'`|
|7|JOIN条件错|`JOIN t2 WHERE id = id`|`JOIN t2 ON t1.id = t2.id`|
|8|比较运算符|`WHERE age == 18`|`WHERE age = 18`|
|9|别名引用错|`WHERE 别名 > 3`|`HAVING 别名 > 3`|
|10|分号遗漏|`SELECT * FROM t`|`SELECT * FROM t;`|
